# CyberLab - Educational Cybersecurity Game

An interactive educational game designed to teach cybersecurity concepts through engaging gameplay. Developed as a graduation project at Al-Yamamah University, College of Engineering and Architecture.

## üéÆ About The Project

CyberLab is an educational cybersecurity game that combines interactive gameplay with fundamental cybersecurity concepts to create an immersive learning experience. The game targets beginners aged 13-30 who are interested in learning about cybersecurity through practical, hands-on scenarios.

### Key Features

- **Four Interactive Levels**: Each level focuses on different cybersecurity threats
  - Level 1: Phishing Email Identification
  - Level 2: Social Engineering Defense
  - Level 3: Malware Detection & Mitigation
  - Level 4: DDoS Attack Analysis & Defense

- **User Authentication System**: Secure login and registration with progress tracking
- **Real-time Feedback**: Immediate feedback on player actions and decisions
- **Progress Tracking**: Monitor achievements, scores, and level completion
- **Educational Resources**: Access to supplementary learning materials
- **Tutorial System**: Comprehensive guides for game mechanics and cybersecurity concepts

## üöÄ Getting Started

### Installation

1. Clone the repository
```bash
git clone https://github.com/Talalmh19/cyberlab.git
```

2. Open the project in Unity
```bash
cd cyberlab
# Open Unity Hub and add the project
```

3. Install required packages
- Unity Authentication
- Unity Cloud Save
- Newtonsoft.Json

4. Configure Unity Services
- Set up Unity Authentication in the Unity Dashboard
- Enable Cloud Save service

### Running the Game

1. Open the main scene in Unity
2. Press the Play button in Unity Editor
3. Or build the game for your target platform

## üìö Game Structure

### Core Systems

- **Authentication System**: Handles user registration and login
- **Data Manager**: Manages saving/loading game progress
- **Level Manager**: Controls level progression and difficulty
- **Feedback System**: Provides educational feedback to players
- **Scoring System**: Tracks player performance and achievements

### Database Schema

The game uses the following data structure:
- **Account**: User credentials and basic information
- **Player**: Game progress, scores, and level data
- **Admin**: Administrative privileges
- **Feedback**: User feedback collection
- **Level**: Level definitions and content
- **Tutorial**: Instructional content
- **EducationalResource**: Supplementary learning materials
